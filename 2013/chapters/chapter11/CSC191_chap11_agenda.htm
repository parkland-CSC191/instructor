4.       Basic idea of Group Functions

a.       The author spends some time trying to explain what the term “aggregation” means. From dictionary.com:

                                                               i.      a group or mass of distinct or varied things, persons, etc.: an aggregation of complainants.

                                                             ii.      collection into an unorganized whole.

                                                            iii.      the state of being so collected.

                                                           iv.      Biology, Ecology. a group of organisms of the same or different species living closely together but less integrated than a society.

b.      A simple function returns one result per row. A group function returns one result per multiple rows

c.       Some of the most widely used group functions are: COUNT, MAX, MIN and AVG

5.       COUNT (pg 390 – 392)

a.       Counts the number of rows that contains the specified thing. To count all rows:
select count(*) from employees;

b.      Let’s say we only want to count the bonuses:
select count(bonus) from employees;

c.       Notice how 1 bonus was left out – COUNT does not include NULL values

d.      select count(*) from employees where bonus is null; 

e.      It is possible to nest a function inside a group function, but only 1 nested function is allowed. For instance, if we want to find the number of distinct categories (from the BOOKS table), what would we do?

f.        Look at the “flawed” example on page 392:
select count(shipdate) from orders where shipdate is null;

6.       MAX & MIN (pg 393 – 394)

a.       Pretty self-explanatory. The big thing to note is that these functions can be used on both numeric and character data

b.      select max(length(category)) from books;

c.       select min(category) from books;

7.       AVG (pg 388 – 390)

a.       Very much like MAX & MIN, but only works on numeric data:
select avg(category) from books;

8.       Combine them together

a.       What does this mean?
select count(*), count(order#), count(*)-count(order#), count(distinct order#), max(order#), avg(order#), min(order#) from orderitems;

9.       GROUP BY (pg 395 – 398)

a.       This is how you answer any subset questions; for example “What is the max retail for each category?”
select category, max(retail) from books group by category;

b.      The examples in this section are very good, hence we will walk through them

c.       Pg 395 – why does this one spawn an error?

                                                               i.      select category, avg(retail-cost) profit from books;

d.      pg 396 – this fixes the error in c

                                                               i.      select category, avg(retail-cost) profit from books group by category;

e.      pg 397 – we now have correct syntax, but are doing extra work for no good reason

                                                               i.      select title, avg(retail-cost) profit from books group by title;

f.        pg 398 – Now to make things more interesting, we want a table join (see 2nd paragraph of page 397)

group by customer#, order#;

10.   HAVING – restricting output (pg 398 – 401)

a.       The big thing with HAVING is that you restrict the output after you have already collected it. Sometimes this is the only way to do it; but always try to restrict output before you gather it.

b.      Note the “flawed” example on page 401:
select category, avg(retail-cost) profit from books group by category having avg(retail-cost) > 15 and category in ('COMPUTER','CHILDREN','BUSINESS');
